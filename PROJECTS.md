## ğŸ¯ How Projects Work

1. **Propose** - Submit project idea via [Project Proposal](.github/ISSUE_TEMPLATE/project_proposal.md)
2. **Approval** - Maintainers review and approve
3. **Setup** - Maintainers create new repository in OS-Portfolio-Projects org
4. **Recruit** - Project opens for contributor applications
5. **Build** - Team collaborates to build the project
6. **Complete** - Project finishes, stays open for improvements
7. **Maintain/Archive** - Ongoing improvements or archival (manual decision)

**Want to contribute?** Browse projects below and apply via their "Join Project" link.

## ğŸ’¡ Propose a Project

Have an idea for a new collaborative data engineering project?

**Steps:**
1. Review [existing projects](#) to avoid duplication
2. Read the [Project Proposal Guide](docs/PROJECT-PROPOSAL-GUIDE.md)
3. Submit proposal via [Project Proposal Template](.github/ISSUE_TEMPLATE/project_proposal.md)

**What makes a good project?**
- Simulates real production scenarios
- Has clear Phase 2 twist (requirements evolution)
- Includes FinOps constraints
- Teaches 3-5 core skills
- Can be built collaboratively (2-10 people)

## â“ FAQ

### Can I contribute to multiple projects?
Yes! Many contributors work on 2-3 projects simultaneously.

### What if a project has no contributors?
The proposer can build solo, or the project goes dormant until interest emerges.

### How do I become a project lead?
Propose a project and volunteer to lead, or take over a dormant project.

### Can projects collaborate?
Yes! Projects can share components, learnings, and even contributors.

### What happens to my contributions when a project archives?
Your work remains credited, public, and usable. Your GitHub graph reflects contributions.
